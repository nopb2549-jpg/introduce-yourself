ระบบการทำงาน

บนหน้า Shop ผู้ใช้สามารถเข้ามาดูรายการสินค้าได้ทั้งหมด แต่ถ้าต้องการสั่งซื้อ จำเป็นต้อง สมัครสมาชิกและล็อกอินก่อน เพื่อความปลอดภัยและเพื่อให้ระบบบันทึกข้อมูลการสั่งซื้อได้อย่างถูกต้อง เมื่อผู้ใช้เลือกสินค้าที่ต้องการ ระบบจะมีตัวเลือกให้ซื้อแบบ ชำระเต็มจำนวน หรือ ซื้อแบบชำระล่วงหน้าบางส่วน (ผ่อนชำระ) หากเลือกแบบผ่อนชำระ ผู้ใช้จะต้องจ่ายเงินล่วงหน้าก่อนจำนวนหนึ่ง จากนั้นเมื่อถึงช่วงใกล้วันส่งของ ระบบจะแจ้งเตือนให้ชำระส่วนที่เหลือก่อนจัดส่งสินค้า ระบบจะ “จำรายการที่เลือกไว้” ให้โดยอัตโนมัติ ทำให้ผู้ใช้สามารถกลับมาดูหรือชำระต่อได้ไม่หลุดรายการ

ปัญหา และ วัตถุประสงค์

ปัญหาที่พบเจอคือ หลายเว็บไม่มีระบบชำระเงินแบบผ่อนชำระ ทำให้ผู้ใช้ที่มีงบจำกัดไม่สามารถสั่งซื้อสินค้าได้ ดังนั้นโปรเจกต์นี้จึงพัฒนาระบบผ่อนชำระที่เปิดโอกาสให้ผู้ใช้ชำระเงินบางส่วนล่วงหน้า และชำระส่วนที่เหลือก่อนส่งสินค้า เพื่อเพิ่มความยืดหยุ่นและตอบโจทย์ผู้ใช้มากขึ้น

วิธีแก้ และ แนวทาง

ฒนาระบบสั่งซื้อที่รองรับการชำระเงินแบบผ่อนชำระ โดยให้ผู้ใช้สามารถเลือกชำระเงินบางส่วนก่อน เพื่อยืนยันการสั่งซื้อ และชำระส่วนที่เหลือในภายหลังเมื่อใกล้ถึงกำหนดจัดส่งสินค้า ภายในระบบจะมีการติดตามยอดที่ชำระแล้วและยอดคงเหลืออย่างชัดเจน รวมถึงแจ้งเตือนให้ผู้ใช้ชำระส่วนที่เหลือก่อนจัดส่งสินค้า เพื่อให้การซื้อขายสะดวกขึ้น ยืดหยุ่นขึ้น และช่วยให้ผู้ใช้ที่มีงบจำกัดสามารถซื้อสินค้าได้ง่ายกว่าเดิม

สิ่งที่เรียนรู้

ได้เรียนรู้การออกแบบหน้า ux และ ui ในการ วาง Flow การใช้งานตั้งแต่ดูสินค้า ตะกร้า Checkout ชำระเงิน พร้อมกับ ระบบแสดงและการจัดสินค้า ดึงข้อมูลสินค้าจากฐานข้อมูลจริง เก็บรูปหลายภาพด้วย JSON คำนวณราคาเต็มและราคามัดจำ แสดงสต๊อกแบบ real-time หรือจะเป็นในส่วนของตะกล้าสินค้า เพิ่ม/ลบสินค้า ปรับจำนวน เลือกวิธีชำระ (มัดจำ / จ่ายเต็ม) ดึงข้อมูลด้วยการ JOIN หลายตาราง และในการจัดระบบออเดอร์แบบเต็มรูปแบบ เรียนรู้การจัดการข้อมูลหลายตารางพร้อมกัน เช่น orders, order_items, products ผ่าน transaction พร้อม logic สำคัญ ชำระเต็ม ให้ หักสต๊อกทันที มัดจำ ให้ หักสต๊อกตอนลูกค้าจายครบ
